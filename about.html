Core Principles
How I Think About Engineering
🏗️
Architecture First
No sprint should start without a clear design decision. I write ADRs, run design reviews, and ensure teams never hit walls mid-execution because the architecture wasn't thought through.
🔬
Production Honesty
Systems aren't real until they handle failure gracefully. I build for idempotency, circuit breaking, and observability — not as afterthoughts, but as first-class requirements.
🧑‍💻
Stay in the Code
I maintain 40–60% direct code contribution even in leadership. Teams need leaders who can review a PR critically, debug at 2am, and have the technical credibility to make hard calls stick.
📈
Grow the Team
Delivery at scale requires distributed decision-making. I invest in Tech Lead capacity, pair programming, and mentorship — because the goal is for the team to not need me for every answer.
🤖
AI as Architecture Tool
I use Claude Code agents and structured prompting to compress architecture cycles — not to replace thinking, but to accelerate it. A well-crafted prompt with role, context, and constraints is an engineering artifact in its own right.
